--- layout: post title: Using WiX (under Votive) without installing onto your TFS build server --- I always prefer to install the minimum amount of software on my build server and instead commit tools into the source repository alongside the source code. This, along with other benefits, means that the tools are versioned alongside the source code and different projects can have different versions of the tools.

To do this with WiX for use on a TFS build server, do the following:

  1. Install WiX on a developer machine.
  2. Copy the files from "C:\Program Files (x86)\MSBuild\Microsoft\WiX\v3.x" to a folder in your source tree.
  3. Copy the files from "C:\Program Files (x86)\Windows Installer XML v3.5\bin" to the same folder.
  4. Edit your wixproj file as a text file.
  5. Add the following lines just under the opening Project tag:
      <PropertyGroup>
        <SolutionRoot Condition="'$(SolutionRoot)' == ''">$(SolutionDir)</SolutionRoot>
        <WixToolPath>$(SolutionRoot)\ThirdParty\WiX\</WixToolPath>
        <WixTargetsPath>$(WixToolPath)Wix.targets</WixTargetsPath>
        <WixTasksPath>$(WixToolPath)wixtasks.dll</WixTasksPath>
      </PropertyGroup>
  6. Check all these files in and build.
